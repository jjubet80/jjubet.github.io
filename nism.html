Unit 1: Network and Information Security Management: History & Definitions

This module is on information security management; history and definitions. the module also focused on the standards in information security and how they affect cyber security.The origin of computer security can be traced back to the time of compatible timesharing system where multiple users could access a system and also see others passwords. information security management is based upon 4 concepts; Confidentiality, Integrity, availability and No-repudiation. Confidentiality is talking about disclosure to unauthorised individuals or entities. it talks about how information should be guarded, who needs a particular information and who should be restricted, what is the implication of this information getting into the wrong hands. Integrity talks about how accurate the information is, how is this information important, how should the information be categorised. Availability is about how the information could be access, how can it be used. Non repudiation is about the ability to track an event, the ability to know when an event has happen to be able to prove when and how it hapened.
Threat and vulnerabilities were discussed. A threat is any unwanted incident that can potentially harm a system or an organisation. vulnerability is a form of weakness to any form of asset. Threat modelling system known as STRIDE was developed by Microsoft categorising threats in to different forms. STRIDE stands for SPOOFING, TAMPERING, REPUDIATION, INFORMATION DISCLOSURE, DENIAL OF SERVICE ELEVATION OF PREVILEGE. spoofing is falsifying data, to gain an illegitimate advantage.Tempering is the ability of the attacker to modify data and render data in a different form. Repudiation is when an attacker can deny what has taken place. Information disclosure is when the attacker has access to confidential information. Denial of service is where legitimate users are being denied access to information. Elevation of privilege is when the attacker is able to grant himself authorised access. Also discussed in this unit is DREAD analysis. DREAD is a risk-assessment tool for security threats. DREAD stands for Damage, Reproduceability, Exploitability, Affected users, Discoverability. Damage assesses the extend of devastation of the attack, Reproduceability assess how easily can it spread, Exploitability asseses how simple it is to launch the attack, Affected user assessesn the users that can be affected and Discoverability assesses the ease of discovering the threat.

Unit 5: Network Tools and Components

In this unit, we learned the various types of networks. A Local Area Network (LAN) is basically most common and similar to the home network. Laptops or desktops with ethernet connections use coaxial cable to connect. These were for many connected over a shared single cable but has now evolved into star topology. In the shared single cable network, a device will communicate to the other computers using a broadcast to every connected computer on the shared network. A communication protocol the CSMA/CD (Carrier Sense with Multiple Access and Collision Detection). Was used to detect where there is a collision and arrange for re-transmitting Other devices and terminologies learnt were switches, mac address, vitiual LANs,wifi networks, CSMA/CA (Carrier Sense with Multiple Access and Collision Avoidance), WEP(Wired equivalent privacy), WPA(WiFi protected Access), modulator/demodulator(Modem), ISDN(Integrated Services Digital Network), ADSL(Asymmetric Digital Subsriber Line) Wide Area Networks( WANs), connects LANs together. It forms the backbone of the internet as involves the routers that switch packets over large networks. The ISP(internet service providers) use routers that provide network to LANs or home networks. Roters have the control pane and the forwarding pane. The control pane maintains route information and the forwarding pane controls the forwarding of packet to destinations. Routers maintain routing tables which can be static or dynamic.static routing tables are manually maintained while dynamic routing table is based on algorithms and rules. It is controlled by the RIP(routing information protocol), EIGRP(Enhanced Interior Gateway Routing Protocol), OSPF(Open Shortest Path First), BGP(Border Gateway Protocol), MPLS (Multiprotocol Label Switching). Other terminologies are GPRS(General Packet Radio Services), GSM(Global system for Mobile Communication) Firewalls are filters usually situated at points that separates a trusted network from an untrusted network. Firewalls can be packet filtering. Identifying legitimate traffic based on the packet attributes. firewalls can used stateful inspection. They maintained an internal state table which tracked details of packets such as transport layer flags and sequence numbers. Other firewalls are application firewalls or web application firewalls (WAF) which inspect packets at higher levels of the network stack, up to the application layer. Next Generation Firewall (NGFW) uses the UTM(unified Threat Management technique) to incorporate several defence mechanism into 1. the NGFW detects and blocks threats using the IPS(Intrusion prevention system), IDS(Intrusion detection system), DPI (Deep Packet Inspection), user authentication etc. Points to note when using firewalls are strong passwords other than default password, security certificates installed for all communications, efficient configuration, restriction on disabling firewalls, regular patching, review of vendor security bulletins, provision for constant power supply, effecting loggin and monitoring etc The history of Network Operating system was also treated. Unit 7: Risks and Standards

This module is on Risk and Standards. A lecture was presented which touch down on 3 major areas. Risk, Business continuity and Disaster recovery and General Data Protection Regulation. In other to properly evaluate risk, Security managers need to collect information from different sources and analyse risk from different perspectives. Collecting information can be in form of a risk register, gathering data from sources such as risk notification mailboxes, threat workshops, business impact assesements, gap analysis, and security audits. Evaluating risk entails assigning them either a numerical (quantitative) or subjective (qualitative) assessment value. Qualitative assessment can be by categorising risk into minor, medium or major. These can be assigned a numerical value later if desired. Quantitive assessments would use statistical or historical data to evaluate and assign a weighting to a risk. After evaluating all the risk gathered, the next step is the decision stage where actions are considered to either tolerate the risk, eliminate the risk, reduce the risk, or transfer the risk to other forms. Mitigating actions that be employed once these decisions are taken include security guards/patrols, alarms systems and CCTV; new processes and procedures, firewalls, IDS, SIEM (security information and event management) solutions.

Business continuity plans and disaster recovery (BC & DR) should form part of the security Architecture. These are ways of managing risk. Business continue ensures plans are in place for minimal disruption of services in the event of an attack or failure. Disaster recovery is a subset of Business continue which ensures systematic renaissance of infrastructure in the event of an attack.

Several frameworks such as TOGAF (The Open Group Architecture Framework), The architecture development method (ADM)and SABSA (Sherwood Applied Business Security Architecture) are models which incorporate BC and DR.

There are two factors that drive the DR plan and design. The Recovery Time Objective (RTO) and The Recovery Point Objective (RPO). RTO determines how quickly the IT infrastructure needs to be recovered in the event of an incident, and RPO determines from what point the system needs to be recovered General Data Protection Regulation GDPR set out guidance for data protection. GDPR is based on 7 principles name:

Lawfulness, Fairness and Transparency.

Purpose Limitation (processing data only for the identified and explicit original.

Data Minimisation - data must be relevant and limited to what is necessary.

Accurate and where possible kept up to date.

Storage Limitation - personal data is only kept for a period consistent with the purpose it was collected for.

Integrity and Confidentiality, data is processed in a manner that ensures appropriate security of the personal data.

Accountability, it is the responsibility of the controller for complying with the principles, and to have appropriate processes and records in place to demonstrate that they have complied.

An important regulation of the GDPR is article 5 (1) (f) which states;

'Processed in a manner that ensures appropriate security of the personal data, including protection against unauthorised or unlawful processing and against accidental loss, destruction or damage, using appropriate technical or organisational measures' Salient points in this is the Technical measures and Organisational measure.
Technical measures include assessment of the legal basis for processing, location of data, categories of data, retention schedules and staff and processor access. Also includes physical” security such as physical locks, physical access controls (locked cabinets in locked rooms for special category data) and secure paper disposal.

Organisational measure include staff training, policies, procedures, documentation and monitoring. Policies and procedures include Access control and Password Management, Asset Management, Business continuity, Bring Your Own Device and Remote Access, Breach Policy, Data retention and Erasure policy, Email Internet Usuage and Archive, Clear desk policy etc

Other standards noted where ISO 27000 family of guides and standards governing security incidence, codes of practice and people process and technology. The Payment Card Industry – Data Security Standard (PCI-DSS), NIST i.e, The National Institute of Standards and Technology, an American non-regulatory agency. The Federal Information Processing Standards (FIPS) . The Control Objectives for IT (COBIT) deals with emphasis on risk management, information security and compliance. Unit 9: System Logging and Forensics

This unit focused on System Logging and Forensic. A lecture was presented which discussed the importants of system logging and forensics. Topic discussed were Logs, security Breaches and Digital Forensics

A definition of logs was given as that which was stated by NIST.

“A log is a record of the events occurring within an organization’s systems and networks. Logs are composed of log entries; each entry contains information related to a specific event that has occurred within a system or network”

It is important to keep track of events as they are hapening as this will help identify and rectify issues when things go wrong. Security systems rely on Logs. There are different types of Logs.

The syslog system is a part of the core system logging on the BSD, Linux operating system.In this system, the logs are store in the /var/log directory. The logs are categorised based on their functions. Logs for system mails, boot up messages, mail servers meassages,kernel messages, device driver loggs, Loggins and Authentication attempts, web servers and database logs etc

UNIX system use the Logrotate which are stored in the /etc/logrotate.conf file
The windows systems use the event viewer

Virtually all log messages consist of a timestamp, a hostname and an application (program name) field, as well as some indication of the status.

Log entries are classified by different types the most important logs are the Warning, Alert, Emmergency, Critical etc. Less important logs are Information, debug. Types are:Error, Information, Warning, Success audit and failure audit for Windows systems, Emergency, Alert, Critical, Notice, Debug for MacOS and Linux systems

Types of Logging tools include

Syslog-ng
Nagios
Snort
Log Analysis Tools discussed are:

ELK(elasticsearch, logstash and kibana)
SIEM(Security Information and Event Management)

Security breaches

This focuses on security incident response plan. This must follow the PDCA mandate – plan, do, check and act. The security incident management plan should also nested with the business continuity plans and disaster recovery

The breach response checklist is required to ensure that breach is properly managed to conform with the ICO recommendation. This checklist should follow these steps: Check what types of data were affected?, Determine what happened?, Determine who was responsible?, Stop any escalation(s), Instigate Business Continuity Plan, Determine whether the ICO needs to be notified, Determine whether affected individuals need to be notified

Digital forensic involves recovering digital evidence from its storage or media and this is done in 4 phases – collection, examination, analysis and reporting. Digital forensics tools can be used for many things which include data acquisition and archiving. Examples of digital forensic tools include FTK (Forensic Toolkit), Caine, Sysinternals, SIEM, Nagios, IDS
